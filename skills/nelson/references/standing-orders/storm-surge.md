# Standing Order: Storm Surge

Do not assign tasks that are too large, too vague, or too interdependent for a single agent to deliver.

**Symptoms:**
- Agent produces no output for extended periods despite being actively working.
- Requests for clarification increase as the agent struggles to define boundaries.
- Partial outputs do not integrate with the larger mission because the task was not properly scoped.

**Remedy:** Break large tasks into deliverables that a single captain can complete within one context window. Each task must have a named output, clear acceptance criteria, and no more than two dependencies. If a captain signals that a task is too large, the admiral must re-decompose it before work continues.
